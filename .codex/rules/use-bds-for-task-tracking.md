Использование `bd` обязательно и строгое:

1. После компакта/суммаризации сессии, если в проекте есть `.beads/`, запускай `bd prime`; если каталога нет, пропускай этот шаг без автогенерации beads-конфигурации.
2. Перед любыми кодовыми изменениями должна быть активная задача в `bd`:
   - выбрать через `bd ready`/`bd show`
   - перевести в `in_progress` через `bd update <id> --status in_progress`
3. Вести прогресс только в `bd`, не использовать markdown TODO/планы как источник статусов.
4. По завершении работы:
   - обновить статус задачи (`done`/`closed` по процессу проекта, обычно `bd close <id>`)
   - выполнить `bd sync`
5. Если работа не завершена — зафиксировать follow-up задачей в `bd`, не оставлять незадокументированные хвосты.
6. Не утверждать, что задача завершена, пока её статус не отражён в `bd`.
